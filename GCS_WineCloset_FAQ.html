Frequently Asked Questions — Meeting Reference · April 21, 2026

About this document: This FAQ covers the most common questions about the GCS Wine Closet QR Tracking System — how it works, what it can and can't do, and what we plan to improve. Brought to the April 21 meeting as a reference for Maggie, Christian, and the wider team.
Day-to-Day Operations
01 What happens if someone scans the wrong QR code by mistake — can it be undone? Operations ▶

There is no automatic undo button, but the fix is quick — a two-step correction using the form itself takes about 30 seconds.

Example — Checked out Wine A when you meant Wine B
  • Scan Wine A again → select Check-In → enter the same quantity → restores the count
  • Scan Wine B → select Check-Out → enter the correct quantity → logs the right transaction

Once the Activity Log is live, both the original error and the correction will appear as rows — giving a clear, honest audit trail rather than hiding the mistake.

Near-term improvement: We can add a "Correction / Mistake" option to the Action field in the form. Selecting it skips any quantity update and just logs a note — preventing a bad submission from affecting the count at all.
02 What stops someone from submitting incorrect quantities? Operations ▶

Right now, the system is trust-based — the form accepts any number. There are three layers of protection we can add:

  • Form restriction (quick win): Set a maximum value of 12 on the Quantity field — no one can accidentally type 100. Go to Quantity question → ... → Restrictions → Number → Less than or equal to → 12.
  • Power Automate alert (moderate): If quantity entered is greater than 6, the flow sends a Teams notification flagging it for review. Not a hard block, but a safety net for unusual entries.
  • Negative quantity guard (important): Currently nothing prevents stock from going below zero. A condition before the update step can catch this and alert rather than proceed.
The best answer long-term is the Activity Log — even if someone submits wrongly, there's a full record to trace and correct. Auditability beats prevention in a small, trusted team.
Access & Permissions
03 Who can see the Microsoft List — is it just Operations or the whole company? Access ▶

Currently the list lives on the Grand Cru Selections SharePoint site and inherits its permissions. That site has 49 members — so visibility is broad by default.

Restricting access to Operations only is straightforward: List Settings → Permissions → Stop inheriting → manually assign access. Takes about 5 minutes and can be done before go-live.

Decision needed: Should this list be Operations-only, or is wider read access desirable? See the Sales question below for context.
04 Can the sales team see current stock levels from their phones? Access ▶

Yes — anyone with SharePoint site access can open the list on any phone browser and see live counts instantly. No app needed.

  • Direct link: Share a bookmark to the list view — one tap and they're in.
  • Filtered view: Create a "Sales View" in Microsoft Lists showing only In Stock wines, sorted by shelf — a clean read-only snapshot perfect for pre-visit checks.
  • Read-only access: Grant sales team View permission only — they can see counts but can't edit anything.
This is a genuine value-add for the sales team — knowing exactly what's in the closet before a client tasting or delivery call.
Technical & Reliability
05 What happens if the flow fails silently — how would we know? Technical ▶

Power Automate automatically emails the flow owner (jcarroll@grandcruselections.com) when a run fails — subject line: "Your flow GCS Wine Inventory Room failed." This is on by default and is the baseline safety net.

How to confirm email alerts are enabled
  • Go to Power Automate → My flows → GCS Wine Inventory Room
  • Click Edit → Settings → confirm "Send email notification for failed runs" is ON

For a more proactive alert, we can add a failure branch using "Configure run after" on each step — if any step fails, an immediate Teams message fires rather than a quiet email. This is on the enhancements list.

06 Does this work without WiFi in the wine room? Technical ▶

No — the form requires an active internet connection to submit. Power Automate runs in the cloud, so there is no offline mode. The QR code will scan and the form will open, but submission will fail without connectivity.

  • Check signal strength in the wine room before go-live — scan a QR and submit a test response from that exact spot.
  • WiFi extender — if the room is in a basement or dead zone, a simple extender solves it.
  • Cell data fallback — staff phones on a data plan don't need office WiFi at all.
Action item: Test connectivity in the wine room before printing labels.
07 What if someone doesn't have a smartphone? Technical ▶

The QR scan step requires a phone camera, so a smartphone is the primary device. Two alternatives if needed:

  • Shared device: A dedicated tablet or phone mounted or stored in the wine room — charged and ready, no personal device required.
  • Direct form URL: Any computer with a browser can access the form directly. Staff type the SKU manually instead of having it pre-filled. A desktop shortcut or browser bookmark makes this a one-click action.

For the GCS team where everyone likely has a smartphone, this is probably not a real blocker — but good to have an answer ready.

Known Limitations & Roadmap
08 What are the known holes in the system right now? Known Issues ▶

Being transparent about current limitations and the plan to address them:

● Fix Before Go-Live
305-Wine Import Incomplete
Only ~10 rows are in the list. Scanning an unlisted wine logs nothing silently.
● Fix Before Go-Live
Quantity Can Go Negative
No guard prevents stock from going below zero if more bottles are checked out than exist.
● Fix Soon
Audit Action Logic
Audit should set quantity absolutely, not add to it. Current flow adds, which corrupts audit entries.
● Fix Soon
No Staff Name Capture
Last Action By is blank. Need a Staff Name field on the form to track who did what.
● Nice to Have
QR Labels Show Stale Counts
Labels show original spreadsheet quantities, not live counts. Cosmetic only — the list is always accurate.
● Nice to Have
Status Doesn't Auto-Update
When quantity hits zero, Status stays "In Stock." Should auto-change to "Out of Stock."
09 What enhancements are planned after go-live? Roadmap ▶
  • Activity Log — separate list recording every scan as a new row, full history of every bottle movement
  • Low Stock Alert — automatic Teams/email notification when a wine drops to 2 bottles or fewer
  • Negative quantity guard — prevents check-outs that would take stock below zero
  • Audit branch — third flow path that sets quantity absolutely for physical counts
  • Auto Out of Stock status — Status field updates automatically when quantity hits zero
  • Correction / Mistake action — form option that skips quantity update, logs a note only
  • Sales read-only view — filtered Microsoft Lists view for the sales team
  • Live QR tool — HTML label generator pulling live data from SharePoint instead of embedded snapshot
